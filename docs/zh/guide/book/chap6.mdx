# 第6章 链表

在开始学习链表`LinkedList`之前，我也看了其它几本书的数据结构讲解，有相当一部分是把数组和链表两种数据结构放在一起讲的。两都都是`线性数据结构`，都可以很方便的存储数据。

**数组**在绝大多数的语言（`Java`、`C#`）中声明一个数组时，都要先声明该数组的长度。在内存中分配的是连续内存，且大小是固定的。在数据插入或者移除一个元素成本都很高，因为需要移动整个结构中的元素。

**链表**存储有序的元素集合，但不同于数组。链表在内存中并不是连续的放置，且可以动态扩展。每个元素由自身和指向一个的引用的指针组合。

### 链表的定义

> **链表 linked list**是一种线性数据结构，其中的每个元素都是一个节点对象，各个节点通过“引用”相连接。引用记录了下一个节点的内存地址，通过它可以从当前节点访问到下一个节点

![](/single-linked-list.jpg)

与数组相比较，链表元素的插入或者添加，并不需要移动其它元素，只需要找到目标位置，调整其它指向下个元素的引用，即可完成对应的操作。现实生活中一个很典型的例子就是火车。链表主要分为以下几类：

- 单向链表
- 双向链表
- 循环链表
- 有序链表

::: tip链表的节点
每个链表的节点(Node)，一般情况下都有两个属性。`val`和`next`。`val`为该节点的值，`next`为指定下一个节点的引用（指针），当节点为最尾时，其`next`指向为null。下面为一个典型的`LinkedNode`代码

``` typescript title="LinkedNode.ts"
export class LinkedNode<T> {
  val: T;
  next: undefined | LinkedNode<T>;
  constructor(v: T) {
    this.val = v;
    this.next = undefined;
  }
}
```
:::

## 单向链表

单向链表是指：链表在连接的过程中，只通过next指针，指向下一个节点，为单向串联链表结构。首个节点称之为头节点，最后一个节点称为尾节点，尾节点的`next`指向为`null`

![](/single-linked-list-pic.jpg)

## 双向链表

与单向链表相比较而言，双向链表在链表节点内部多了一个指针指向前一个节点。也就是说一个节点有`next`指针和`prev`指针。在遍历上，可以正向遍历也可以反向遍历。
